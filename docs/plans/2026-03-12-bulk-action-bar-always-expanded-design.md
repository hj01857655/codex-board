# 批量操作栏去除“更多操作”并默认展示高级按钮（设计）

## 背景与目标
用户希望在批量操作控制台中直接点击“启用/禁用/删除/清空”等按钮，不需要先点击“更多操作”。

## 方案对比
1. 方案 A（推荐）：移除“更多操作/展开”入口，高级操作区始终展示。
   - 优点：操作直达，最符合需求。
   - 缺点：占用更多空间。
2. 方案 B：保留展开入口但默认展开。
   - 优点：可折叠；
   - 缺点：仍存在入口，与“不要更多操作”冲突。
3. 方案 C：将高级按钮上移到摘要栏。
   - 优点：更紧凑；
   - 缺点：改动更大，布局可能拥挤。

## 采用方案
采用方案 A：移除“更多操作/展开”入口，高级按钮始终展示。

## 设计要点
- 移除“展开/收起”按钮与“更多操作”按钮。
- 始终渲染高级操作区（测试/启用/禁用/删除/清空）。
- 保留标题、副标题、已选数量与并发信息。

## 影响范围
- 仅涉及前端 UI：`src/components/bulk/BulkActionBar.tsx`。
- 不改动业务逻辑与接口。

## 测试与验证
- UI 手动检查：批量操作栏默认显示高级按钮；不再出现“更多操作/展开”。
